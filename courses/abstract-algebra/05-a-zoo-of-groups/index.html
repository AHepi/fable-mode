A Zoo of Groups

What you'll be able to do
  • List at least four distinct examples of groups and identify the set and operation for each
  • Verify that a given example meets all four group axioms (closure, associativity, identity, inverses)
  • Distinguish finite groups from infinite groups by counting elements
  • Recognize when a group is abelian (commutative) and when it is not
  • Read and build a small Cayley table for a finite group

Once you have a key, you start seeing locks everywhere. The definition of a group — a set equipped with an operation that satisfies four rules — looked abstract in module 04, a list of conditions without a face. But definitions are lenses: the moment one clicks into focus, the world fills up with examples you had walked past without recognizing.

This module is the gallery. We will visit six specimens: some made of ordinary numbers, one made of the square root of −1-1−1, and a few familiar objects that fail the test. For each we ask the same four questions from module 04, and the answers come back differently every time — same skeleton, different flesh.

Recall the four axioms from module 04. A group is a set GGG together with an operation ∗*∗ (a rule that combines any two elements of GGG to get another element of GGG) satisfying:

  1. Closure: a∗ba * ba∗b is always in GGG.
  2. Associativity: (a∗b)∗c=a∗(b∗c)(a * b) * c = a * (b * c)(a∗b)∗c=a∗(b∗c) for all a,b,ca, b, ca,b,c in GGG.
  3. Identity: there is a special element eee in GGG — the identity, the element that leaves every partner unchanged — with a∗e=e∗a=aa * e = e * a = aa∗e=e∗a=a for every aaa.
  4. Inverses: every element aaa in GGG has an inverse a−1a^{-1}a−1 in GGG with a∗a−1=a−1∗a=ea * a^{-1} = a^{-1} * a = ea∗a−1=a−1∗a=e.

A group where a∗b=b∗aa * b = b * aa∗b=b∗a for every pair is called abelian (after Niels Henrik Abel, the Norwegian mathematician who used this symmetry to prove that the quintic equation has no formula). We say the operation is commutative.

The integer line: (Z,+)(\mathbb{Z}, +)(Z,+)

The integers — …,−2,−1,0,1,2,…\ldots, -2, -1, 0, 1, 2, \ldots…,−2,−1,0,1,2,… — under ordinary addition.

  • Closure: add two integers and you get an integer. ✓\checkmark✓
  • Associativity: (1+2)+3=1+(2+3)(1 + 2) + 3 = 1 + (2 + 3)(1+2)+3=1+(2+3). Integer addition has been associative since you learned it. ✓\checkmark✓
  • Identity: 000. Adding zero to anything leaves it unchanged: a+0=aa + 0 = aa+0=a. ✓\checkmark✓
  • Inverses: the inverse of aaa is −a-a−a. Indeed a+(−a)=0a + (-a) = 0a+(−a)=0. ✓\checkmark✓

So (Z,+)(\mathbb{Z}, +)(Z,+) is a group. It is abelian (a+b=b+aa + b = b + aa+b=b+a always) and infinite — it has no last element in either direction.

Note on notation. Because the operation here is honest addition, we write +++, not the generic ∗*∗. Each group in this module keeps its own natural operation — +++ for integers, ×\times× for products — rather than replacing every symbol with the abstract placeholder.

The clock group: (Zn,+)(\mathbb{Z}_n, +)(Zn​,+)

From module 02 we know Zn\mathbb{Z}_nZn​ (the nnn-hour clock): the set {0,1,2,…,n−1}\{0, 1, 2, \ldots, n-1\}{0,1,2,…,n−1} where addition wraps around at nnn. For instance, on the 5-hour clock (Z5\mathbb{Z}_5Z5​), we have 3+4=23 + 4 = 23+4=2 (not 777, since 7−5=27 - 5 = 27−5=2).

  • Closure: adding two numbers from {0,1,…,n−1}\{0, 1, \ldots, n-1\}{0,1,…,n−1} and wrapping at nnn always lands back in {0,1,…,n−1}\{0, 1, \ldots, n-1\}{0,1,…,n−1}. ✓\checkmark✓
  • Associativity: inherited from ordinary integer addition. ✓\checkmark✓
  • Identity: 000. Zero added to anything on the clock leaves it where it is. ✓\checkmark✓
  • Inverses: the inverse of aaa is n−an - an−a (from module 04: a+(n−a)=n≡0a + (n - a) = n \equiv 0a+(n−a)=n≡0). In Z5\mathbb{Z}_5Z5​, the inverse of 333 is 222, since 3+2=5≡03 + 2 = 5 \equiv 03+2=5≡0. ✓\checkmark✓

So (Zn,+)(\mathbb{Z}_n, +)(Zn​,+) is a group for every n≥1n \geq 1n≥1. It is abelian and finite — it has exactly nnn elements. We write the size of a group as ∣G∣|G|∣G∣, so ∣Zn∣=n|\mathbb{Z}_n| = n∣Zn​∣=n.

The nonzero rationals: (Q×,×)(\mathbb{Q}^\times, \times)(Q×,×)

Let Q×\mathbb{Q}^\timesQ× denote the set of all nonzero rational numbers (fractions pq\frac{p}{q}qp​ with p,qp, qp,q integers and q≠0q \neq 0q=0, and the fraction itself nonzero).

  • Closure: the product of two nonzero fractions is a nonzero fraction. ✓\checkmark✓
  • Associativity: multiplication of numbers is associative. ✓\checkmark✓
  • Identity: 111. ✓\checkmark✓
  • Inverses: the inverse of pq\frac{p}{q}qp​ is qp\frac{q}{p}pq​, which is a nonzero rational so long as p≠0p \neq 0p=0. ✓\checkmark✓

This is an infinite abelian group.

Why exclude zero? If 000 were in the set, it would have no inverse: there is no rational xxx with 0×x=10 \times x = 10×x=1. One missing inverse is enough to disqualify the whole set. So we throw out 000 to make the axioms work.

Two things that are NOT groups

Not every set with an operation earns the title.

The integers under multiplication, (Z,×)(\mathbb{Z}, \times)(Z,×). Multiplication on Z\mathbb{Z}Z has closure, associativity, and identity (111). But inverses fail: the inverse of 222 under multiplication would need to be 12\frac{1}{2}21​, which is not an integer. A single missing inverse sinks the whole structure.

The natural numbers under addition, (N,+)(\mathbb{N}, +)(N,+), where N={0,1,2,3,…}\mathbb{N} = \{0, 1, 2, 3, \ldots\}N={0,1,2,3,…}. Here identity exists (000) and closure holds. But there is no inverse for 111: the inverse would need to be −1-1−1, which is not a natural number. The natural numbers do not reach back far enough.

These non-examples sharpen what the axioms protect. The group axioms are a checklist for a reason: each one excludes something that would otherwise cause the structure to misbehave.

5. In any Cayley table of a group, every element appears exactly once in each row and exactly once in each column. Explain, in words, why a row cannot have a repeated element.

Show solution

Suppose row aaa has a repeated entry: that means a∗b=a∗ca * b = a * ca∗b=a∗c for two different elements b≠cb \neq cb=c. But every element has an inverse a−1a^{-1}a−1 in the group. Apply it on the left of both sides:

a−1∗(a∗b)=a−1∗(a∗c).a^{-1} * (a * b) = a^{-1} * (a * c).a−1∗(a∗b)=a−1∗(a∗c).

By associativity, (a−1∗a)∗b=(a−1∗a)∗c(a^{-1} * a) * b = (a^{-1} * a) * c(a−1∗a)∗b=(a−1∗a)∗c, which gives e∗b=e∗ce * b = e * ce∗b=e∗c, so b=cb = cb=c. That contradicts b≠cb \neq cb=c. So no row can have a repeated entry. The same argument (using right-inverses) handles columns.

This is why Cayley tables look like Latin squares — the “every element once” pattern is forced by the group axioms, not chosen by hand.

Whether these two groups are “really the same” in a structural sense is precisely the question module 10 (homomorphisms and isomorphisms) is built to answer.

Recap

The abstract definition of a group from module 04 is not a single idea — it is a template that many different objects fit. Integers under addition, clocks, two signs under multiplication, the fourth roots of unity, nonzero fractions: all of them satisfy the same four axioms (closure, associativity, identity, inverses), and the Cayley table captures the whole structure of a finite group in a single grid.

Every group we have seen so far is abelian: a∗b=b∗aa * b = b * aa∗b=b∗a. That is not a theorem — it is a coincidence of the examples we have visited. A paper triangle, a triangular hole, and six ways to put one inside the other will end it.
