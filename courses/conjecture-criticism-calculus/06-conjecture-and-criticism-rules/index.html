Conjecture and Criticism: the Building Rules

What you'll be able to do
  • State the (Conj) transition rule as a premise/conclusion pair and identify the problem premise.
  • Explain why (Conj) is disabled when the problem set is empty.
  • State the (Crit) transition rule and identify the warrant it requires.
  • Apply both rules to a small epistemic state and produce the resulting state and attack graph.
  • Explain why "refuted" is a computed status, not an erasure, and why the no-deletion principle matters.

The system we have built so far is a very detailed snapshot: a tuple S=(A,Π,addr,att,W)S = (A, \Pi, \mathrm{addr}, \mathrm{att}, W)S=(A,Π,addr,att,W) holding artifacts, problems, an addressing relation, an attack graph, and a warrant map (module 4). The attack graph is well-formed exactly when every edge carries a registered warrant (module 5). But a snapshot, however precise, does not move. This module gives the system its first two engines of motion — two transition rules that rewrite a state SSS into a successor state S′S'S′.

Transitions in this calculus are written S⇝S′S \rightsquigarrow S'S⇝S′: “state SSS can become state S′S'S′ in one step.” Each rule is a premise/conclusion pair: the premises are what must hold in SSS before the rule fires; the conclusion says what gets added to produce S′S'S′. By design, every transition only adds — nothing is ever removed. That principle is not a convenience; it is one of the load-bearing commitments of the calculus, and we will come back to it.

The two rules this module owns are (Conj), which introduces a new conjecture, and (Crit), which introduces a new criticism along with its warranted attack edge.

(Conj): Conjecture introduction

A conjecture is an artifact introduced to address a problem (module 3). Every conjecture needs a problem to answer — there is nothing to conjecture about without one. The conjecture–criticism loop from module 1 begins with a problem precisely because problems are what drive inquiry forward; (Conj) makes that structural rather than motivational.

Here is the picture before the formalism: you have a current state SSS with at least one problem π\piπ sitting in Π\PiΠ. The conjecture operator γ\gammaγ takes π\piπ and the whole current state SSS and produces a new artifact a=γ(π,S)a = \gamma(\pi, S)a=γ(π,S) — the conjecture. That artifact gets registered: added to AAA, linked to π\piπ via addr\mathrm{addr}addr, and equipped with its commitment interface I(a)I(a)I(a).

The bridge: the premises below are exactly “there is a problem to address” and “the operator produces something.” The conclusion adds that something and records where it is aimed.

Rule (Conj)

π∈Πa=γ(π,S)S′=(A∪{a},  Π,  addr∪{(a,π)},  att,  W)\frac{\pi \in \Pi \qquad a = \gamma(\pi, S)}{S' = \bigl(A \cup \{a\},\; \Pi,\; \mathrm{addr} \cup \{(a,\pi)\},\; \mathrm{att},\; W\bigr)}S′=(A∪{a},Π,addr∪{(a,π)},att,W)π∈Πa=γ(π,S)​

In words: if there is a problem π\piπ in the current problem set, and γ\gammaγ produces a new artifact aaa in response to it and to the current state, then S′S'S′ is SSS with aaa added to the artifact pool and the pair (a,π)(a,\pi)(a,π) added to the addressing relation. The attack relation att\mathrm{att}att and the warrant map WWW are untouched — the conjecture arrives without attacking anything.

The disabled case. When Π=∅\Pi = \emptysetΠ=∅ — no problems registered — the rule has no first premise to satisfy. (Conj) is disabled. The calculus cannot conjecture in a vacuum; inquiry begins with a problem, not with an artifact searching for one. This is constraint D1 of the source note made structural: problems are primitive, and the engine stalls without them.

Check yourself: why is (Conj) disabled when $\Pi = \emptyset$?

The rule’s first premise is π∈Π\pi \in \Piπ∈Π. If Π=∅\Pi = \emptysetΠ=∅, there is no π\piπ to satisfy it. The rule is a conditional: if the premises hold, fire; since the premise cannot hold, the rule cannot fire. It is not that conjecture is forbidden — it is that there is nothing to conjecture about. The calculus encodes “problems first” as a structural constraint, not a policy suggestion.

(Crit): Criticism introduction

A criticism is an artifact kkk that attacks another artifact aaa, and every attack edge must carry a warrant (module 5). The (Crit) rule adds kkk to the artifact pool, draws the attack edge (k,a)(k, a)(k,a) in att\mathrm{att}att, and registers the warrant www in WWW.

The picture: you have a current state SSS containing some artifact aaa. Someone constructs a criticism kkk — itself an artifact in the sense of module 3, with content c(k)c(k)c(k) and interface I(k)I(k)I(k) — together with a valid warrant www for the claim that kkk attacks aaa. (The warrant is either demonstrative — packaging a failing verdict — or argumentative, as module 5 explains.) The rule fires, and S′S'S′ contains kkk, the new edge, and the warrant’s registration.

The bridge: the premises say aaa exists and kkk arrives with proof-of-attack. The conclusion wires everything together.

Rule (Crit)

a∈Ak an artifactw a valid warrant for (k,a)S′=(A∪{k},  Π,  addr,  att∪{(k,a)},  W∪{(k,a)↦w})\frac{a \in A \qquad k \text{ an artifact} \qquad w \text{ a valid warrant for } (k,a)}{S' = \bigl(A \cup \{k\},\; \Pi,\; \mathrm{addr},\; \mathrm{att} \cup \{(k,a)\},\; W \cup \{(k,a) \mapsto w\}\bigr)}S′=(A∪{k},Π,addr,att∪{(k,a)},W∪{(k,a)↦w})a∈Ak an artifactw a valid warrant for (k,a)​

In words: if aaa is already in the artifact pool and kkk is presented together with a valid warrant www justifying the attack (k,a)(k,a)(k,a), then S′S'S′ extends SSS by adding kkk to AAA, the edge (k,a)(k,a)(k,a) to att\mathrm{att}att, and the entry (k,a)↦w(k,a) \mapsto w(k,a)↦w to the warrant map WWW.

Notice that kkk is itself an artifact — content plus a commitment interface (module 3). This is the artifact-uniformity principle at work: every criticism is itself an artifact and hence attackable. A criticism does not arrive as a bare verdict (”aaa is wrong”); it arrives as a thing with structure that can be addressed, committed to, and attacked. That is what “whatever can attack can be attacked” means in the formalism.

Check yourself: what would make a warrant invalid, blocking (Crit) from firing?

A warrant www for the edge (k,a)(k,a)(k,a) must satisfy one of the two conditions from module 5: it is demonstrative (packages a failing verdict V(κ,c(a))=failV(\kappa, c(a)) = \text{fail}V(κ,c(a))=fail with a validity node ν(κ)\nu(\kappa)ν(κ) asserting the test is sound) or argumentative (an explanatory artifact whose commitments assert aaa‘s deficiency). If www is neither — if kkk simply claims aaa is wrong without registering a warrant of either type — the well-formedness condition is not met and (Crit) does not fire. Every attack edge in the system carries a warrant; the rule enforces this at the point of introduction.

Nothing is ever deleted

Both rules only add. (Conj) adds an artifact and an addressing pair. (Crit) adds an artifact, an attack edge, and a warrant entry. No rule in this calculus — not (Conj), not (Crit), and not any of the rules to come — removes an artifact, a problem, an edge, or a warrant entry from the state.

This is the no-deletion principle, and it is worth sitting with rather than skipping past.

When a criticism kkk succeeds in attacking a conjecture aaa, nothing happens to aaa in the artifact pool. aaa is still there, in AAA, with its content, its interface, its addressing relation, and every warrant that referenced it. What changes is aaa‘s status — a computed quantity that the adjudication rule (Adj) will assign in module 8, by running the grounded extension over the current attack graph. “Refuted” means: given the current attack graph, the adjudicator assigns aaa the status refuted. It does not mean: aaa was erased.

The distinction is not pedantic. If deletion happened, reinstatement could not: once an artifact is gone, a criticism of its criticism has nothing to revive. The system would lose its memory of what was proposed. But because the system keeps everything and recomputes statuses from the current graph, a new attack on kkk — a criticism of the criticism — can change the graph so that the adjudicator now assigns aaa the status accepted. That is reinstatement, and it is a theorem, not a promise (module 8 will prove it). The theorem only makes sense if the artifact aaa was never removed.

Refuted is a computed status, not an erasure. The artifact persists; the label is a function of the current attack graph.

This is constraint D8 of the source note — “refutation is demotion, not annihilation” — built directly into the transition rules by giving them no delete operation.

This is a lean but well-formed state: one open problem, nothing else.

The premise π0∈Π\pi_0 \in \Piπ0​∈Π holds. 2. Suppose someone proposes applying (Crit) to add an attack edge (k2,a1)(k_2, a_1)(k2​,a1​) but offers no warrant. Is the resulting state well-formed? What does the (Crit) rule say?

Show solution

No. The third premise of (Crit) requires “a valid warrant www for (k2,a1)(k_2, a_1)(k2​,a1​).” Without a warrant, the premise is not satisfied and the rule does not fire. If somehow the edge were added despite this — treating the premise as advisory rather than required — the resulting state would be not well-formed: the §3.2 formation rule states that every attack edge must carry a registered warrant. The rule’s premise and the well-formedness condition enforce the same constraint from two directions: (Crit) will not produce a state with an unwarranted edge, and any state with an unwarranted edge fails the formation check.

Show solution

A second criticism k2k_2k2​ would target k1k_1k1​: its warrant w2w_2w2​ would be valid for the edge (k2,k1)(k_2, k_1)(k2​,k1​). Applying (Crit) with a=k1a = k_1a=k1​, k=k2k = k_2k=k2​, w=w2w = w_2w=w2​ gives:

The attack graph has three nodes and two directed edges: k2→k1→a1k_2 \to k_1 \to a_1k2​→k1​→a1​. This is the classic reinstatement chain: k1k_1k1​ attacks a1a_1a1​, k2k_2k2​ attacks k1k_1k1​. Nothing has been deleted. All three artifacts persist. Whether a1a_1a1​ is now accepted, refuted, or suspended depends on the grounded extension — the question module 7 answers. (Spoiler: in this chain, with k2k_2k2​ unattacked, a1a_1a1​ will turn out accepted.)

4. (Refuted ≠ deleted.) A student argues: “If k1k_1k1​ successfully attacks a1a_1a1​, we should remove a1a_1a1​ from AAA to keep the state tidy. What does it matter that a refuted artifact stays in the pool?” Give two reasons the calculus gives for keeping a1a_1a1​.

Show solution

Reason 1: Reinstatement requires the artifact to exist. If a1a_1a1​ is removed when k1k_1k1​ attacks it, then a later criticism of k1k_1k1​ — say k2k_2k2​ attacks k1k_1k1​ with a valid warrant — has nothing to reinstate. The calculus proves (module 8) that refuting a criticism restores its target. That theorem is only possible because the target was never deleted. Deletion makes reinstatement impossible by construction.

Reason 2: “Refuted” is a computed status, not a fact about the artifact’s content. The label refuted is assigned by the adjudicator (Adj) based on the current attack graph: specifically, a1a_1a1​ is refuted iff some member of the grounded extension attacks it. If the graph changes — because k1k_1k1​ is itself attacked and moves out of the grounded extension — the label can change too. A status is a function of the graph at a moment; it says nothing permanent about a1a_1a1​‘s content. Deleting a1a_1a1​ would be treating a graph-relative label as a verdict about the artifact itself, which is precisely the confusion the no-deletion principle blocks.

5. (Stretch.) Can (Conj) and (Crit) be applied in the reverse order — (Crit) before (Conj) — starting from S0=(∅,{π0},∅,∅,∅)S_0 = (\emptyset, \{\pi_0\}, \emptyset, \emptyset, \emptyset)S0​=(∅,{π0​},∅,∅,∅)? Explain by checking each rule’s premises.

Show solution

No. (Crit)‘s first premise is a∈Aa \in Aa∈A: there must already be some artifact aaa to attack. In S0S_0S0​, A=∅A = \emptysetA=∅, so no such aaa exists and (Crit) cannot fire. (Conj) must go first to place at least one artifact in AAA. This is not a policy choice: it is a consequence of the rule’s premises. The ordering (Conj) → (Crit) is forced by the structure of the rules when starting from an empty artifact pool.

More generally, (Crit) can fire before (Conj) only if the state already contains artifacts — brought in by a prior (Conj) step or by some other means. The rules impose a logical dependency, not a temporal one: criticism requires something to criticize.

Recap

Two transition rules now grow epistemic states: (Conj) adds a conjecture that addresses a problem — and is disabled whenever Π=∅\Pi = \emptysetΠ=∅, making “problems first” a structural fact — while (Crit) adds a criticism together with its warranted attack edge, enforcing the warrant requirement from module 5 at the moment of introduction. Both rules only add; the no-deletion principle means “refuted” is always a computed status assigned by the adjudicator over the current attack graph, never an erasure of the artifact itself. The payoff of holding all this in one place: a growing, fully warranted, non-deleting state is exactly the structure the grounded extension can read unambiguously — every arrow a committed position, every node still in the room.
