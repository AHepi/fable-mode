Artifacts and Problems

What you'll be able to do
  • State Definition 3.2 for artifact and problem, identifying each component of the tuples
  • Explain what the commitment interface $I(a)$ represents and why it is finite
  • Construct a small artifact with a two-commitment interface and verify a verdict against it
  • Explain why conjectures, criticisms, and even the system's own rules are all artifacts
  • Distinguish an artifact's content $c(a)$ from its commitment interface $I(a)$

Every noun in this calculus is going to need a place to stand. In module 2 we gave commitments their footing — a test paired with a budget, returning one of three verdicts. Now the question is: what are the things those commitments test?

The answer turns out to be the same for everything: conjectures, criticisms, the standards that judge them, even the rules governing the calculus itself. They are all artifacts. That uniformity is not a convenience — it is the engine behind the system’s most important property, and we will keep returning to it.

From pairs to everything else

Picture the simplest possible content: a string of symbols, a sentence claiming something. On its own it is just ink. It becomes a player in the calculus when you attach to it a finite roster of commitments — the tests it is officially answerable to. Strip away all the philosophy, and an artifact is exactly that: content plus a list.

A problem is a different shape. It has a description (what the problem is asking), a set of criteria (what a solution would have to satisfy), and a provenance record (where the problem came from). The criteria are not arbitrary: they are commitment schemas — templates that can be instantiated against any candidate artifact to produce actual commitments. A problem’s criteria grow as inquiry proceeds; the artifact’s commitment interface is fixed at registration time (though more commitments can be added by introducing a new artifact).

These two shapes — artifact and problem — are the two nouns the whole calculus runs on. The next definition gives them precisely.

Definition 3.2 (Artifact, Problem)

An artifact is a=(c(a), I(a))a = (c(a),\, I(a))a=(c(a),I(a)): a content c(a)c(a)c(a) together with a finite commitment interface I(a)I(a)I(a), a finite set of commitments (each in the sense of Definition 3.1) that aaa is answerable to.

A problem is π=(dπ, Critπ, provπ)\pi = (d_{\pi},\, \mathrm{Crit}_{\pi},\, \text{prov}_{\pi})π=(dπ​,Critπ​,provπ​): a description dπd_{\pi}dπ​, a growing finite set of criteria Critπ\mathrm{Crit}_{\pi}Critπ​ (commitment schemas instantiable against candidate artifacts), and a provenance record provπ\text{prov}_{\pi}provπ​.

Artifacts are untyped beyond this: conjectures, criticisms, aesthetic standards, problem criteria, experimental reports, and the calculus’s own rules are all artifacts. This uniformity is what makes “whatever can attack can be attacked” enforceable.

Bridge: reading the tuples

Let’s decode each piece.

For an artifact a=(c(a),I(a))a = (c(a), I(a))a=(c(a),I(a)):

  • c(a)c(a)c(a) is the content — the thing the artifact actually says or proposes. Think of it as the payload: a conjecture about bread chemistry, a criticism of that conjecture, a rule saying “attacks must carry warrants.” All of these have content.
  • I(a)I(a)I(a) is the commitment interface — a finite set of commitments {κ1,κ2,…}\{\kappa_1, \kappa_2, \ldots\}{κ1​,κ2​,…}, each of the form κ=(τ,β)\kappa = (\tau, \beta)κ=(τ,β) from Definition 3.1, that aaa is officially answerable to. Running V(κi,c(a))V(\kappa_i, c(a))V(κi​,c(a)) for each κi∈I(a)\kappa_i \in I(a)κi​∈I(a) is the formal meaning of “testing aaa against its interface.”
  • The interface being finite matters: it keeps the question “does aaa pass all its commitments?” decidable in bounded time.

For a problem π=(dπ,Critπ,provπ)\pi = (d_{\pi}, \mathrm{Crit}_{\pi}, \text{prov}_{\pi})π=(dπ​,Critπ​,provπ​):

  • dπd_{\pi}dπ​ is the description — the prose statement of what is being asked (“Why does ice expand instead of contracting like most solids?”).
  • Critπ\mathrm{Crit}_{\pi}Critπ​ is the criteria set — a finite (and growing) collection of commitment schemas. A schema is like a commitment with a blank where the content goes; plug in a candidate artifact aaa and you get an actual commitment κa\kappa_{a}κa​ whose verdict tells you whether aaa meets that criterion.
  • provπ\text{prov}_{\pi}provπ​ is the provenance — a record of how this problem arose: from a failed verdict, from two surviving rivals, from a reach event, and so on. We will meet these triggers when we reach the (Spawn) rule in a later module; for now, provenance is the problem’s birth certificate.

The crucial asymmetry: a problem’s criteria set can grow (as inquiry proceeds, new demands may be placed on any solution), while an artifact’s commitment interface is fixed at the moment the artifact enters the system. New demands on an artifact are handled by introducing a new artifact that answers to an extended interface.

The artifact-uniformity principle: Nothing in the definition distinguishes a conjecture from a criticism from a methodological standard. They all have content and a commitment interface. This means any artifact can be the target of an attack (provided the attacker has a warrant), and any artifact can serve as attacker. “Whatever can attack can be attacked” follows directly — not as a philosophical commitment added from outside, but as a consequence of the uniform definition.

Check yourself: what does $I(a) = \emptyset$ mean for an artifact?

An artifact with I(a)=∅I(a) = \emptysetI(a)=∅ has an empty commitment interface — no commitments it is answerable to. No commitment can return fail\text{fail}fail on it, because there are no commitments to run. Such an artifact has no attack surface: it cannot be criticised via a demonstrative warrant (which requires a failing verdict), and in the measures we will define in module 9, it will turn out to be not criticizable. An empty interface is not a virtue — it means the artifact makes no testable commitment, which is as close to epistemically inert as the system allows.

Worked example

Let’s build a concrete artifact and a concrete problem and see how they fit the definitions.

Artifact. Let aaa be the conjecture: “Yeast fermentation produces carbon dioxide, which causes bread dough to rise.”

We register it with a two-commitment interface I(a)={κ1,κ2}I(a) = \{\kappa_1, \kappa_2\}I(a)={κ1​,κ2​}:

  • κ1=(τ1,β1)\kappa_1 = (\tau_1, \beta_1)κ1​=(τ1​,β1​) where τ1\tau_1τ1​ checks whether the content mentions a specific gas and its role. Running V(κ1,c(a))V(\kappa_1, c(a))V(κ1​,c(a)) on the sentence above returns pass\text{pass}pass — it does identify CO2\text{CO}_2CO2​.
  • κ2=(τ2,β2)\kappa_2 = (\tau_2, \beta_2)κ2​=(τ2​,β2​) where τ2\tau_2τ2​ checks whether the content predicts a measurable volumetric increase in dough under standard fermentation conditions. Running V(κ2,c(a))V(\kappa_2, c(a))V(κ2​,c(a)) returns pass\text{pass}pass if the content quantifies the expansion, fail\text{fail}fail if it does not. Suppose our sentence does not quantify it: then V(κ2,c(a))=failV(\kappa_2, c(a)) = \text{fail}V(κ2​,c(a))=fail.

So aaa passes κ1\kappa_1κ1​ and fails κ2\kappa_2κ2​. It has a partial attack surface — κ2\kappa_2κ2​ failing is exactly the kind of foothold a demonstrative criticism needs (more on that structure in module 5). The artifact is a pair (c(a),{κ1,κ2})(c(a), \{\kappa_1, \kappa_2\})(c(a),{κ1​,κ2​}); that is everything the system needs to track it.

Problem. Let π\piπ be the problem: “Why does bread dough rise during baking?”

As a tuple: π=(dπ,Critπ,provπ)\pi = (d_{\pi}, \mathrm{Crit}_{\pi}, \text{prov}_{\pi})π=(dπ​,Critπ​,provπ​) where:

  • dπd_{\pi}dπ​ is the description above.
  • Critπ\mathrm{Crit}_{\pi}Critπ​ initially contains one criterion schema: a solution must identify a specific physical or chemical mechanism. Instantiated against candidate aaa, this schema produces a commitment whose test checks for mechanism-identification — aaa satisfies it.
  • provπ\text{prov}_{\pi}provπ​ records that this problem was posed by the user at the start of the session (provenance type: initial problem).

Later, after seeing two surviving rival conjectures, Critπ\mathrm{Crit}_{\pi}Critπ​ could grow to include a discrimination criterion: any solution must make a prediction that distinguishes it from the rival. The problem’s criteria set grows; the existing artifacts’ interfaces do not change automatically. A new artifact addressing the expanded criteria would need a correspondingly extended interface.

Check yourself: if someone offers a criticism $k$ of the conjecture $a$ above, what kind of object is $k$?

kkk is itself an artifact: a tuple (c(k),I(k))(c(k), I(k))(c(k),I(k)) with its own content and its own commitment interface. The content c(k)c(k)c(k) might say “the conjecture aaa fails κ2\kappa_2κ2​ because it gives no quantitative prediction.” The interface I(k)I(k)I(k) consists of whatever commitments kkk is answerable to — at minimum, the commitments required by the warrant kkk carries for its attack on aaa.

This is the artifact-uniformity principle in action. kkk is not a special “criticism object” with different rules — it is an artifact, the same kind of object as aaa, which means it can in turn be attacked by another artifact carrying a suitable warrant. The criticism is itself criticizable.

The variation kernel: a signpost

Definition 3.2 in the source includes one more item alongside artifact and problem: the variation kernel μ(⋅∣a)\mu(\cdot \mid a)μ(⋅∣a), a samplable distribution over bounded edits of c(a)c(a)c(a). This object belongs to the demarcation and modifiability story, which module 9 develops in full. We name it here because it appears in the definition; we will not use it until then.

Exercises

1. Consider a conjecture bbb with content “all metals conduct electricity” and commitment interface I(b)={κ}I(b) = \{\kappa\}I(b)={κ}, where κ\kappaκ tests whether the content makes a universal claim about a named material class. What is V(κ,c(b))V(\kappa, c(b))V(κ,c(b))?

Show solution

The test κ\kappaκ checks for a universal claim about a named material class. The content “all metals conduct electricity” makes exactly such a claim — it uses “all” and names the class “metals.” So V(κ,c(b))=passV(\kappa, c(b)) = \text{pass}V(κ,c(b))=pass.

Note: this verdict says bbb satisfies that particular commitment. It says nothing about whether the claim is true in the world — the commitment is testing a formal property of the content (does it make a universal claim?), not doing physics.

2. You are given a problem π′\pi'π′ with description “Which alloys conduct electricity at room temperature?” and an initial Critπ′\mathrm{Crit}_{\pi'}Critπ′​ containing one schema: a solution must name at least one specific alloy. Conjecture bbb from Exercise 1 does not name any alloy. Describe what V(κb,c(b))V(\kappa_{b}, c(b)) V(κb​,c(b)) looks like when the schema in Critπ′\mathrm{Crit}_{\pi'}Critπ′​ is instantiated against bbb.

Show solution

Instantiating the schema against bbb produces a commitment κb\kappa_{b}κb​ whose test checks whether c(b)c(b)c(b) names at least one specific alloy. The content “all metals conduct electricity” names no specific alloy — “metals” is a category, not an alloy. So V(κb,c(b))=failV(\kappa_{b}, c(b)) = \text{fail}V(κb​,c(b))=fail.

This is the mechanism by which a problem’s criteria can reject a candidate: the criterion schema, when instantiated against bbb, produces a failing verdict. bbb is answerable to this commitment (once it is added to I(b)I(b)I(b) by an appropriate transition — but that mechanism belongs to module 6). What matters here is the shape: criteria are schemas, schemas instantiate against artifacts to produce commitments, commitments produce verdicts.

3. Explain, in one paragraph, why the statement “this proof is incorrect” (presented as a criticism) must also be an artifact in the sense of Definition 3.2.

Show solution

The definition of an artifact makes no distinction by type: anything in the system is an artifact if it has content and a commitment interface. The statement “this proof is incorrect” has content (c(k)c(k)c(k) = the claim of incorrectness) and it can be paired with a commitment interface — for instance, a commitment that tests whether the stated error location actually exists in the target proof. That commitment produces a verdict on c(k)c(k)c(k).

Because it is an artifact, “this proof is incorrect” can itself be attacked by another artifact carrying a warrant. If someone shows that the identified error is not actually an error, that is a criticism of the criticism — and the machinery for handling it is exactly the same as for any other attack. This is the artifact-uniformity principle: the system does not special-case criticisms, and so criticisms are always criticizable. That is not an add-on; it follows from the definition.

Show solution

The proposal “add κ3\kappa_3κ3​ to aaa‘s interface, with V(κ3,c(a))=failV(\kappa_3, c(a)) = \text{fail}V(κ3​,c(a))=fail” is a claim — it has content. For it to function as a criticism in the calculus, it must be an artifact: a tuple (c(k),I(k))(c(k), I(k))(c(k),I(k)) where c(k)c(k)c(k) states the proposed extension and the failing verdict, and I(k)I(k)I(k) contains at least the commitments that the warrant type requires. A demonstrative warrant, for instance, requires that the commitment κ3\kappa_3κ3​ be packaged into the content of kkk, together with a trace establishing V(κ3,c(a))=failV(\kappa_3, c(a)) = \text{fail}V(κ3​,c(a))=fail and a validity node asserting that κ3\kappa_3κ3​ is a sound and relevant test for aaa (we will make this precise in module 5). The proposal to modify aaa‘s interface is not an administrative act outside the calculus — it enters as an artifact, is subject to the same warrant discipline, and can be attacked like any other artifact.
