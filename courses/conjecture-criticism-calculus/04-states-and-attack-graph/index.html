Module 4 of 11

Epistemic States and the Attack Graph

What you'll be able to do
  • State Definition 3.3 and name each of the five components of an epistemic state S
  • Read att⊆A×A and addr⊆A×Π as binary relations on finite sets
  • Draw and read a directed attack graph from a given att, identifying nodes, edges, and who attacks whom
  • Construct a small epistemic state from a plain-English description of a critical exchange

What do you have to record to capture inquiry at a single instant — not just a list of ideas, but the full mathematical structure of who exists, what problems they address, and who is currently targeting whom? A debate can freeze in the middle; a scientific exchange can pause mid-volley. Whatever the snapshot is, it must be precise enough that two people looking at it agree on every fact: who is in the room, what each artifact claims to answer, and every attack that has been lodged. That is exactly the question Definition 3.3 answers. A single tuple, five components, and you have a complete formal record of inquiry at any moment in time.

Directed graphs: nodes and arrows

Before the definition, you need one piece of structure the calculus depends on. You have seen relations as sets of ordered pairs — from your set-theory background, a relation RRR on a set AAA is just a subset R⊆A×AR \subseteq A \times AR⊆A×A, meaning a collection of pairs (x,y)(x, y)(x,y) drawn from AAA. A directed graph is that idea given a picture: draw each element of AAA as a dot (a node) and each pair (x,y)∈R(x, y) \in R(x,y)∈R as an arrow from xxx to yyy (a directed edge).

The key word is directed: the arrow from xxx to yyy is not the same as the arrow from yyy to xxx. The edge (x,y)(x, y)(x,y) goes one way. If both (x,y)(x,y)(x,y) and (y,x)(y,x)(y,x) are in RRR you draw two arrows, one each way. If neither is, you draw none.

In this calculus, the set of nodes is the artifact set AAA, and the relation is the attack relation. An edge (k,a)∈att(k, a) \in \mathrm{att}(k,a)∈att carries one meaning: k attacks a. That is all. No edge means no attack, and no attack means no claim from kkk against aaa — at least not yet.

Reading directed graphs is a skill you will use for the rest of this course, so practice it on a small example before the formal definition.

Small example. Suppose we have three artifacts: a1a_1a1​, a2a_2a2​, a3a_3a3​. The attack relation is:

att={(a2,a1),  (a3,a2)}.\mathrm{att} = \{(a_2, a_1),\; (a_3, a_2)\}.att={(a2​,a1​),(a3​,a2​)}.

As a directed graph:

a_3  →  a_2  →  a_1

Read it left to right: a3a_3a3​ attacks a2a_2a2​, and a2a_2a2​ attacks a1a_1a1​. No arrow points at a3a_3a3​; nothing attacks it. Nothing points from a1a_1a1​ outward; it does not attack anyone.

There are only two edges and three nodes — but already the structure tells a story. a1a_1a1​ is under attack, a2a_2a2​ is both attacking and under attack, and a3a_3a3​ is attacking but itself unattacked. As the course unfolds, this layout will determine exactly which artifacts stand and which fall. For now, simply read the graph: an arrow from kkk to aaa means kkk attacks aaa.

Definition 3.3 — Epistemic state

With directed graphs in hand, here is the definition the module is built around.

Definition (Epistemic State)

An epistemic state is a five-tuple

S=(A, Π, addr, att, W)S = (A,\, \Pi,\, \mathrm{addr},\, \mathrm{att},\, W)S=(A,Π,addr,att,W)

where:

  • AAA is a finite set of artifacts,
  • Π\PiΠ is a finite set of problems,
  • addr⊆A×Π\mathrm{addr} \subseteq A \times \Piaddr⊆A×Π is the addressing relation — the set of pairs (a,π)(a, \pi)(a,π) such that artifact aaa is offered as a response to problem π\piπ,
  • att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A is the attack relation — the set of ordered pairs (k,a)(k, a)(k,a) such that artifact kkk attacks artifact aaa,
  • WWW is the warrant map — a function assigning to each edge (k,a)∈att(k, a) \in \mathrm{att}(k,a)∈att a warrant justifying that attack.

(Definition 3.3)

Decoding each component

The definition packs five objects into one line. Here is what each one is doing, and why each slot has to exist.

AAA — the artifact set. You need a roster: a place to register every piece of content that is actively in play. Every conjecture, every criticism, every problem criterion, every experimental report — lives in AAA as an artifact a=(c(a),I(a))a = (c(a), I(a))a=(c(a),I(a)) (the content plus the commitment interface you saw in module 3). AAA is finite at any moment; the transition rules can add to it but never delete.

Π\PiΠ — the problem set. Inquiry is always about something; without a list of open questions, you cannot say what the artifacts are competing to answer. Every open question the system is tracking is a problem π=(dπ,Critπ,provπ)\pi = (d_\pi, \mathrm{Crit}_\pi, \mathrm{prov}_\pi)π=(dπ​,Critπ​,provπ​): a description, a growing set of criteria, and a provenance record. Like AAA, the set Π\PiΠ grows but never shrinks.

addr⊆A×Π\mathrm{addr} \subseteq A \times \Piaddr⊆A×Π — addressing. Knowing that an artifact and a problem both exist is not enough; you need to record which artifacts are put forward as answers to which problems. A pair (a,π)∈addr(a, \pi) \in \mathrm{addr}(a,π)∈addr says that artifact aaa is put forward as addressing problem π\piπ. This relation tracks which ideas are competing to answer which questions. Not every artifact addresses every problem; not every problem has an artifact addressing it yet. The relation is declared explicitly — an artifact does not automatically address a problem just by mentioning it.

att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A — the attack graph. This is the heart of the state. Each pair (k,a)∈att(k, a) \in \mathrm{att}(k,a)∈att says that artifact kkk attacks artifact aaa. Read it as a directed graph: nodes are the artifacts in AAA, and each pair is an arrow. In the “map of arrows” picture: k→ak \to ak→a means kkk attacks aaa. Nothing more, nothing less. The structure of this graph — who has attackers, who is defending someone, who is unattacked — is what the adjudication rule (later) will turn into a verdict on every artifact.

WWW — the warrant map. An attack edge without a justification is a bare accusation; to be contentful, every edge must carry a reason that can itself be examined and attacked. Every (k,a)∈att(k, a) \in \mathrm{att}(k,a)∈att must be backed by a justification: a warrant. The map WWW records what that justification is for each edge. There are two kinds of warrant, demonstrative and argumentative, and the full treatment belongs to module 5. For now, just note that WWW is part of the tuple — every arrow in the attack graph comes tagged with a reason, and the system enforces this.

Worked example: building a state

Here is a small critical exchange and the epistemic state it induces.

The scenario. A student conjectures that a certain recursive function terminates on all inputs. A critic argues that the student’s analysis overlooks a boundary case. A defender then argues that the critic’s boundary-case objection misreads the base case of the recursion. Three artifacts, one problem.

Building AAA. We have three artifacts:

  • a1a_1a1​ — the conjecture (the termination claim with its analysis).
  • a2a_2a2​ — the criticism (pointing to the alleged boundary-case gap).
  • a3a_3a3​ — the defense (arguing that the boundary-case objection misreads the base case).

So A={a1,a2,a3}A = \{a_1, a_2, a_3\}A={a1​,a2​,a3​}.

Building Π\PiΠ. There is one problem: π1\pi_1π1​ — “Does this recursive function terminate on all inputs?”

So Π={π1}\Pi = \{\pi_1\}Π={π1​}.

Building addr\mathrm{addr}addr. Only a1a_1a1​ is put forward as an answer to π1\pi_1π1​; the criticism and the defense are attacks, not direct answers to the termination question.

addr={(a1,π1)}.\mathrm{addr} = \{(a_1, \pi_1)\}.addr={(a1​,π1​)}.

Building att\mathrm{att}att. The criticism a2a_2a2​ attacks the conjecture a1a_1a1​. The defense a3a_3a3​ attacks the criticism a2a_2a2​.

att={(a2,a1),  (a3,a2)}.\mathrm{att} = \{(a_2, a_1),\; (a_3, a_2)\}.att={(a2​,a1​),(a3​,a2​)}.

Building WWW. Each edge carries a warrant. W(a2,a1)W(a_2, a_1)W(a2​,a1​) is the warrant justifying a2a_2a2​‘s attack on a1a_1a1​ — say, the specific commitment that a1a_1a1​‘s analysis fails on the base case. W(a3,a2)W(a_3, a_2)W(a3​,a2​) is the warrant justifying a3a_3a3​‘s attack on a2a_2a2​ — the claim that a2a_2a2​ misidentifies what the base case says. (Module 5 will open WWW up and show the two warrant types in full.)

The state.

S=({a1,a2,a3},  {π1},  {(a1,π1)},  {(a2,a1), (a3,a2)},  W).S = \bigl(\{a_1, a_2, a_3\},\; \{\pi_1\},\; \{(a_1, \pi_1)\},\; \{(a_2, a_1),\,(a_3, a_2)\},\; W\bigr).S=({a1​,a2​,a3​},{π1​},{(a1​,π1​)},{(a2​,a1​),(a3​,a2​)},W).

The graph.

a_3  →  a_2  →  a_1    (a_1 addresses π_1)

a3a_3a3​ is unattacked. a2a_2a2​ is attacked (by a3a_3a3​). a1a_1a1​ is attacked (by a2a_2a2​). The conjecture is under attack, but its attacker is itself under attack. What happens to a1a_1a1​ in light of this? That is the question module 7 answers with the grounded extension. For now, we have the snapshot — the state — written down exactly.

Check yourself: reading a state

Consider the epistemic state S=(A,Π,addr,att,W)S = (A, \Pi, \mathrm{addr}, \mathrm{att}, W)S=(A,Π,addr,att,W) where:

A={p,q,r,s},Π={π},A = \{p, q, r, s\}, \quad \Pi = \{\pi\},A={p,q,r,s},Π={π}, addr={(p,π), (q,π)},att={(r,p), (s,r)}.\mathrm{addr} = \{(p, \pi),\, (q, \pi)\}, \quad \mathrm{att} = \{(r, p),\, (s, r)\}.addr={(p,π),(q,π)},att={(r,p),(s,r)}.

Before reading on: Draw the attack graph. Which artifacts are addressing π\piπ? Which artifacts are under attack? Which artifact is unattacked?

Answer.

Attack graph:

s  →  r  →  p        q  (no edges involving q or s as targets)
  • Artifacts addressing π\piπ: ppp and qqq. Both are competing answers to the same problem.
  • ppp is under attack (by rrr). rrr is under attack (by sss).
  • qqq is not under attack — no pair in att\mathrm{att}att has qqq as second element.
  • sss is unattacked — no pair in att\mathrm{att}att has sss as second element.

Note that qqq addresses π\piπ but is completely unattacked. ppp also addresses π\piπ but is currently under fire.

Exercises

Exercise 1 (mechanical). Write out the epistemic state SSS for the following scenario in full tuple notation. There are two artifacts: aaa (a conjecture) and kkk (a criticism of aaa). There is one problem π\piπ, which aaa addresses. kkk attacks aaa.

Show solution A={a,k},Π={π},addr={(a,π)},att={(k,a)}.A = \{a, k\}, \quad \Pi = \{\pi\}, \quad \mathrm{addr} = \{(a, \pi)\}, \quad \mathrm{att} = \{(k, a)\}.A={a,k},Π={π},addr={(a,π)},att={(k,a)}.

WWW assigns a warrant to the single attack edge (k,a)(k, a)(k,a).

S=({a,k},  {π},  {(a,π)},  {(k,a)},  W).S = \bigl(\{a, k\},\; \{\pi\},\; \{(a,\pi)\},\; \{(k, a)\},\; W\bigr).S=({a,k},{π},{(a,π)},{(k,a)},W).

Attack graph:

k  →  a   (a addresses π)

kkk is unattacked. aaa is under attack. kkk does not address π\piπ — it is a criticism, not a proposed answer to the problem.

Exercise 2 (mechanical). Given att={(b,a), (c,b), (b,c)}\mathrm{att} = \{(b, a),\, (c, b),\, (b, c)\}att={(b,a),(c,b),(b,c)} on A={a,b,c}A = \{a, b, c\}A={a,b,c}:

(a) Draw the directed attack graph. (b) List all artifacts that are under attack (i.e., appear as the second element of some pair in att\mathrm{att}att). (c) List all artifacts that attack at least one other artifact. (d) Is any artifact both attacking and under attack?

Show solution

(a) Edges: b→ab \to ab→a, c→bc \to bc→b, b→cb \to cb→c.

     ↗ c ↘
a ← b  ←↗

More precisely: an arrow from bbb to aaa; an arrow from ccc to bbb; an arrow from bbb to ccc. So bbb and ccc have arrows going back and forth between them.

(b) Under attack: aaa (attacked by bbb), bbb (attacked by ccc), ccc (attacked by bbb). All three are targets of some edge.

(c) Attacking at least one other: bbb (attacks aaa and ccc), ccc (attacks bbb). So bbb and ccc.

(d) Yes: both bbb and ccc are simultaneously attacking and under attack. bbb attacks ccc and is attacked by ccc; ccc attacks bbb and is attacked by bbb. This is a cycle, and cycles lead to suspended status in the adjudicator — which module 7 handles.

Note: aaa is under attack but does not attack anyone. It is a purely passive target in this state.

Exercise 3 (conceptual). Suppose two artifacts a1a_1a1​ and a2a_2a2​ both appear in AAA and both address the same problem π\piπ — so (a1,π)∈addr(a_1, \pi) \in \mathrm{addr}(a1​,π)∈addr and (a2,π)∈addr(a_2, \pi) \in \mathrm{addr}(a2​,π)∈addr. Is it required that one attacks the other? Explain why or why not using the definition of att\mathrm{att}att.

Show solution

No. Addressing the same problem does not automatically create an attack edge. The attack relation att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A is built from pairs that are explicitly registered; an artifact does not attack another just by existing or by competing for the same problem.

Two conjectures can both address π\piπ and be completely unconnected in the attack graph — they are rivals in the sense of competing for the same problem, but neither has yet produced a warranted argument against the other. The calculus keeps them both in AAA, both in addr\mathrm{addr}addr, and simply records that no attack edge exists between them. (Module 6 will introduce the (Crit) rule, which is the only way to register a new attack; module 11 will show that surviving rivalry without an attack between rivals is itself a trigger for a new problem.)

This is one place where the formal system is careful: rivalry is not the same as attack. Rivalry is captured by the structure of addr\mathrm{addr}addr; attack is captured by att\mathrm{att}att. They are separate relations for a reason.

Exercise 4 (conceptual). The definition requires att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A, meaning both the attacker and the attacked must be artifacts already in AAA. Explain in one or two sentences why this constraint matters. What would go wrong if att\mathrm{att}att could contain a pair (k,a)(k, a)(k,a) where k∉Ak \notin Ak∈/A?

Show solution

Every edge in att\mathrm{att}att must connect two nodes that actually exist in the artifact set AAA. If k∉Ak \notin Ak∈/A, then kkk is not a registered artifact in this state: it has no content c(k)c(k)c(k), no commitment interface I(k)I(k)I(k), and no warrant map entry — the system has no record of it. An attack from a non-artifact is not a well-formed attack; it cannot carry a warrant, and the adjudicator has nothing to work with on kkk‘s side.

Requiring att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A is the formal statement that only registered artifacts can participate in the attack graph. This also means that adding a new criticism to the system requires first adding it to AAA — which is exactly what the (Crit) transition rule does (module 6).

Exercise 5 (stretch). A directed graph on AAA is called a tournament if for every two distinct nodes a,b∈Aa, b \in Aa,b∈A, exactly one of (a,b)(a,b)(a,b) or (b,a)(b,a)(b,a) is in att\mathrm{att}att. Can an attack graph in this calculus be a tournament? What would that mean for the system, and is there any formal reason it could not happen?

Show solution

Yes, an attack graph can in principle be a tournament. The definition of att\mathrm{att}att imposes no restriction on which pairs are included beyond att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A — both (a,b)(a,b)(a,b) and (b,a)(b,a)(b,a) can simultaneously be in att\mathrm{att}att (a mutual attack), or exactly one, or neither.

A tournament attack graph would mean: every pair of artifacts is in some attack relation — no two artifacts ignore each other. That is an unusual but not forbidden configuration. In a tournament with an odd number of nodes, every artifact is attacked by at least one other, and no artifact is unattacked. The grounded extension (module 7) would compute a fixed point from this structure; how populated that extension is depends on the specific tournament’s structure.

The one formal constraint that would limit this in practice is the warrant requirement: every edge in att\mathrm{att}att must carry a warrant registered in WWW (and must satisfy the formation rules of module 5). Whether a tournament’s full set of edges can all carry valid warrants is a question about the specific content of the artifacts involved — it is not ruled out by the definition of SSS alone.

Recap

An epistemic state S=(A,Π,addr,att,W)S = (A, \Pi, \mathrm{addr}, \mathrm{att}, W)S=(A,Π,addr,att,W) is a precise snapshot of inquiry: who is in the room (AAA and Π\PiΠ), what each artifact claims to answer (addr\mathrm{addr}addr), who is attacking whom (att\mathrm{att}att), and on what grounds each attack rests (WWW). The attack relation att⊆A×A\mathrm{att} \subseteq A \times Aatt⊆A×A is a directed graph — nodes are artifacts, arrows are attacks, and an arrow from kkk to aaa means exactly one thing: kkk attacks aaa. That “map of arrows” is the structure the adjudicator will read to assign each artifact a status. The warrant map WWW — what makes every arrow an argued position rather than a bare accusation — is what module 5 opens up next.
