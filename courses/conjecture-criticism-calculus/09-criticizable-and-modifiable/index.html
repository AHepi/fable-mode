Demarcation: Criticizable and Modifiable

What you'll be able to do
  • Define the variation kernel µ(·|a) and explain what it samples.
  • State what Pr[event] > 0 means in the context of a sampled edit.
  • Define the battery B(a) and battery-equivalence a′ ≈_B a.
  • Classify a variant as battery-equivalent or battery-inequivalent, with justification.
  • State the definitions of crit(a), mod(a), and "epistemically active."
  • Explain why criticizability replaces (rather than merely refines) Popperian falsifiability.

Two artifacts survive adjudication. Call them a1a_1a1​ and a2a_2a2​. From the grounded extension GGG alone, both are accepted — equal partners in the system’s current best picture. But something nags: a2a_2a2​ can be rephrased in a hundred different ways that all pass every test identically, while a1a_1a1​, when you change even a small piece, breaks against the battery. Are they really equal? And a third artifact appears: a3a_3a3​ has no commitment interface at all — nothing to test, nothing to attack. Should the system treat it as a live conjecture?

The previous eight modules built the apparatus — artifact a=(c(a),I(a))a = (c(a), I(a))a=(c(a),I(a)) from module 3, verdicts from module 2, the grounded extension from module 7, adjudication and reinstatement from module 8. What that apparatus does not yet say is which artifacts count as genuine participants in inquiry, and in what sense one explanation is more committed than another. That is this module’s work: a two-part demarcation criterion that replaces falsifiability as the boundary between the live and the inert — and widens that boundary to include mathematics, philosophy, and the arts alongside empirical science.

To get there, you need one tool the course hasn’t introduced yet: a small, precise slice of probability. It takes four paragraphs.

A minimal probability primer

A probability distribution over a set XXX is an assignment of a number Pr⁡[x]∈[0,1]\Pr[x] \in [0,1]Pr[x]∈[0,1] to each element x∈Xx \in Xx∈X, with the numbers summing to 1. Think of it as a random-choice device: when you “sample from the distribution,” you get one element of XXX, with each element appearing as often as its assigned probability.

An event is a subset E⊆XE \subseteq XE⊆X. The probability of the event is Pr⁡[E]=∑x∈EPr⁡[x]\Pr[E] = \sum_{x \in E} \Pr[x]Pr[E]=∑x∈E​Pr[x] — the total weight assigned to the elements in EEE.

The one condition that matters here: Pr⁡[E]>0\Pr[E] > 0Pr[E]>0 means the event is possible — the sampler can produce some element in EEE. It does not mean EEE is likely or common; it means EEE is not ruled out with certainty. Contrast Pr⁡[E]=0\Pr[E] = 0Pr[E]=0: no element in EEE is ever produced. For a finite distribution, Pr⁡[E]=0\Pr[E] = 0Pr[E]=0 means no element of EEE has any weight, so it simply never outputs anything in EEE.

That is the entire probability toolkit this module needs. No continuous distributions, no expectations, no statistics — just: a random-edit machine, an event, and the question of whether the event can happen at all. The random-edit machine becomes the variation kernel μ(⋅∣a)\mu(\cdot \mid a)μ(⋅∣a) introduced next — the formal object that samples bounded edits of an artifact — and Pr⁡[E]>0\Pr[E] > 0Pr[E]>0 is precisely the condition modifiability will turn on.

The variation kernel

An artifact a=(c(a),I(a))a = (c(a), I(a))a=(c(a),I(a)) has content c(a)c(a)c(a). You can imagine making small changes to that content: swap a word, perturb a coefficient, swap one explanatory step for another. The variation kernel μ(⋅∣a)\mu(\cdot \mid a)μ(⋅∣a) is the formal object that captures this: it is a probability distribution over bounded edits of c(a)c(a)c(a).

Think of it as the random-edit machine for aaa. When you sample a′∼μ(⋅∣a)a' \sim \mu(\cdot \mid a)a′∼μ(⋅∣a), you get some modified version of aaa‘s content. The edits are bounded — they stay within reach of the original, rather than ranging over all of Σ∗\Sigma^*Σ∗ — and the distribution is samplable, meaning the sampler can actually produce outputs.

The variation kernel was named in module 3 as part of Definition 3.2 but deferred there. It enters the calculus properly now, because it is exactly what the modifiability definition needs.

The battery and battery-equivalence

Before defining criticizability and modifiability, there is one more object to build: the battery of an artifact.

An artifact aaa has two sources of test obligations. First, its own commitment interface I(a)I(a)I(a): the finite set of commitments aaa answers to directly (module 3 — every artifact carries its own finite set of commitments it must pass). Second, any problem π\piπ that aaa addresses (in the sense of module 3: (a,π)∈addr(a, \pi) \in \mathrm{addr}(a,π)∈addr) carries a growing set of criteria Critπ\mathrm{Crit}_\piCritπ​ — commitment schemas that, when instantiated against aaa, become further obligations.

Bridge to the definition. So the battery B(a)B(a)B(a) is the union of those two sources: aaa‘s own commitments, plus the instantiated criteria of every problem aaa addresses. It is the full set of tests the system currently knows aaa must face.

Definition 3.5 (Battery and battery-equivalence)

Let aaa be an artifact in epistemic state SSS. The battery B(a)B(a)B(a) is the set consisting of aaa‘s own commitments together with the instantiated criteria of all problems aaa addresses via addr\mathrm{addr}addr.

Write a′≈Baa' \approx_B aa′≈B​a (battery-equivalent) if a′a'a′ and aaa receive identical verdicts across every commitment in B(a)B(a)B(a): for each κ∈B(a)\kappa \in B(a)κ∈B(a), V(κ,c(a′))=V(κ,c(a))V(\kappa, c(a')) = V(\kappa, c(a))V(κ,c(a′))=V(κ,c(a)).

Battery-equivalence is the formal answer to “same explanation, possibly reworded.” Two artifacts that agree on every verdict across B(a)B(a)B(a) are, from the battery’s perspective, indistinguishable — they commit to exactly the same claims about every test the system can currently run.

Equivalent vs. inequivalent: a worked example

This distinction is load-bearing — module 10 (hard-to-vary) depends on it — so it deserves a careful worked example.

Setup. Let aaa be the artifact whose content is: “Objects accelerate toward each other with force F=Gm1m2/r2F = Gm_1m_2 / r^2F=Gm1​m2​/r2.” Suppose the battery B(a)B(a)B(a) contains five commitments: tests against planetary orbit data, pendulum timing, tidal patterns, projectile parabolas, and a Cavendish torsion-balance measurement.

Now consider two variants sampled from μ(⋅∣a)\mu(\cdot \mid a)μ(⋅∣a).

Variant a1′a'_1a1′​: a rewording. Content: “The gravitational attraction between two masses varies inversely as the square of the distance separating them.”

Is a1′≈Baa'_1 \approx_B aa1′​≈B​a? Check each commitment in B(a)B(a)B(a). The rewording expresses the same mathematical law — same force, same dependence on mass and distance. Every test in B(a)B(a)B(a) that aaa passes, a1′a'_1a1′​ passes, with the same verdict. Every test aaa fails (if any), a1′a'_1a1′​ fails identically. So V(κ,c(a1′))=V(κ,c(a))V(\kappa, c(a'_1)) = V(\kappa, c(a))V(κ,c(a1′​))=V(κ,c(a)) for all κ∈B(a)\kappa \in B(a)κ∈B(a).

Conclusion: a1′≈Baa'_1 \approx_B aa1′​≈B​a. This is a battery-equivalent variant — linguistically different, informationally the same explanation.

Variant a2′a'_2a2′​: a genuine modification. Content: “Objects accelerate toward each other with force F=Gm1m2/r2.01F = Gm_1m_2 / r^{2.01}F=Gm1​m2​/r2.01.”

Is a2′≈Baa'_2 \approx_B aa2′​≈B​a? The exponent has changed from 2 to 2.01. Precision tests — especially the Cavendish torsion balance and planetary orbit data — are sensitive to this. Running commitment κorbit\kappa_{\text{orbit}}κorbit​ against c(a2′)c(a'_2)c(a2′​) yields a different verdict than running it against c(a)c(a)c(a): the predictions diverge at the precision of the test. So V(κorbit,c(a2′))≠V(κorbit,c(a))V(\kappa_{\text{orbit}}, c(a'_2)) \neq V(\kappa_{\text{orbit}}, c(a))V(κorbit​,c(a2′​))=V(κorbit​,c(a)).

Conclusion: a2′̸≈Baa'_2 \not\approx_B aa2′​≈B​a. This is a battery-inequivalent variant. The battery can distinguish it from aaa.

The key takeaway: ≈B\approx_B≈B​ screens out mere stylistic variation. For the purposes of everything that follows, a rewording is the same artifact. Genuine novelty requires the battery to be able to tell the difference.

Definition 3.5 (Criticizability and modifiability)

Criticizable. An artifact aaa is criticizable, written crit(a)\mathrm{crit}(a)crit(a), if its commitment interface is nonempty:

crit(a)  ⟺  I(a)≠∅\mathrm{crit}(a) \iff I(a) \neq \emptysetcrit(a)⟺I(a)=∅

Bridge. The commitment interface I(a)I(a)I(a) is the finite set of commitments aaa directly carries (module 3). If I(a)=∅I(a) = \emptysetI(a)=∅, no commitment is directly attached to aaa — so no attack of the form “run this test, it fails” is possible. aaa has no attack surface. Criticizability says simply: there is at least one test that aaa claims to pass.

Modifiable. An artifact aaa is modifiable, written mod(a)\mathrm{mod}(a)mod(a), if the variation kernel can produce an inequivalent variant with positive probability:

mod(a)  ⟺  Pr⁡a′∼μ(⋅∣a) ⁣[a′̸≈Ba]>0\mathrm{mod}(a) \iff \Pr_{a' \sim \mu(\cdot \mid a)}\!\left[a' \not\approx_B a\right] > 0mod(a)⟺Pra′∼μ(⋅∣a)​[a′≈B​a]>0

Bridge. The variation kernel μ(⋅∣a)\mu(\cdot \mid a)μ(⋅∣a) is the random-edit machine for aaa. The event {a′̸≈Ba}\{a' \not\approx_B a\}{a′≈B​a} is the set of all sampled edits that produce a battery-inequivalent variant — something the battery can actually distinguish from aaa. The condition >0> 0>0 says this event is possible: the kernel can produce a genuine alternative, not just a rewording. If the probability were 0, every edit of aaa would be battery-equivalent to it — the artifact’s content space, as far as the battery is concerned, is a single point.

Epistemically active. An artifact aaa is epistemically active if both crit(a)\mathrm{crit}(a)crit(a) and mod(a)\mathrm{mod}(a)mod(a) hold.

Check yourself. Artifact bbb has I(b)=∅I(b) = \emptysetI(b)=∅. Is bbb criticizable? Is it epistemically active? Why or why not?

Show answer

bbb is not criticizable: I(b)=∅I(b) = \emptysetI(b)=∅ means crit(b)\mathrm{crit}(b)crit(b) is false. Since criticizability is one of the two conditions for epistemic activity, bbb is not epistemically active regardless of whether mod(b)\mathrm{mod}(b)mod(b) holds. An artifact with no commitment interface has no attack surface — nothing to test and nothing to refute.

Demarcation: why this replaces falsifiability

Popper’s falsifiability criterion — which marked a theory as scientific if it could in principle be refuted by observation — required that some commitment be observation-valued: tied to an empirical test. This ruled out mathematics, philosophy, aesthetic standards, and much else.

Criticizability requires only I(a)≠∅I(a) \neq \emptysetI(a)=∅. A commitment κ∈I(a)\kappa \in I(a)κ∈I(a) is a test program paired with a resource budget (module 2 — a verdict is V(κ,c)∈{pass,fail,overrun}V(\kappa, c) \in \{\text{pass}, \text{fail}, \text{overrun}\}V(κ,c)∈{pass,fail,overrun}). Nothing in that definition says the test must be empirical. A mathematical proof obligation is a commitment. A logical consistency check is a commitment. A constraint from a formal standard in aesthetics is a commitment. All three give rise to a nonempty I(a)I(a)I(a), and thus to crit(a)\mathrm{crit}(a)crit(a).

The source note is explicit: “Empirical falsifiability is the special case of crit\mathrm{crit}crit in which some commitment is observation-valued.” Falsifiability is not a rival to criticizability — it is a special instance of it, the case where some test in I(a)I(a)I(a) is tied to an observable outcome. The demarcation simply generalizes: mathematics, philosophy, and the arts are admitted, not by special pleading, but because their commitments satisfy I(a)≠∅I(a) \neq \emptysetI(a)=∅ on exactly the same footing as empirical commitments.

Modifiability adds the second arm: it rules out artifacts that are locally unreachable — where every sampled edit produces only battery-equivalent rewrites, so the system can generate no genuine variant to compare or compete. An artifact with Pr⁡a′∼μ(⋅∣a)[a′̸≈Ba]=0\Pr_{a' \sim \mu(\cdot \mid a)}[a' \not\approx_B a] = 0Pra′∼μ(⋅∣a)​[a′≈B​a]=0 occupies a solitary point in the variation landscape; the calculus has no surface to push against.

Together, crit(a)\mathrm{crit}(a)crit(a) and mod(a)\mathrm{mod}(a)mod(a) mark the epistemically active artifacts: those that can be attacked and meaningfully varied. This is the demarcation — not “could an experiment falsify it?” but “does it have an attack surface, and is there genuine room to move?”

4. (Conceptual — crit≠\mathrm{crit} \neqcrit= falsifiability) A formal mathematical proof aaa carries one commitment κ\kappaκ: a logical-validity check (the test program verifies each step follows from the preceding ones by the stated inference rules). Is aaa criticizable? Is it falsifiable in Popper’s sense? Explain the difference.

Show solution

aaa is criticizable: I(a)={κ}≠∅I(a) = \{\kappa\} \neq \emptysetI(a)={κ}=∅, so crit(a)\mathrm{crit}(a)crit(a) holds. The commitment κ\kappaκ is a test program that checks logical validity — it has a fixed budget (the computation terminates or overruns), and it returns pass, fail, or overrun against the proof’s content.

aaa is not falsifiable in Popper’s sense. Popper’s criterion requires the possibility of an empirical observation that could refute the theory. A formal proof makes no observation-valued predictions; it could not be refuted by running an experiment.

The difference: criticizability requires only I(a)≠∅I(a) \neq \emptysetI(a)=∅ — any test suffices, empirical or not. Falsifiability requires that at least one test be tied to observable reality. This is the sense in which falsifiability is a special case of criticizability: when some κ∈I(a)\kappa \in I(a)κ∈I(a) is observation-valued, criticizability implies falsifiability; but many artifacts (proofs, formal standards, aesthetic criteria) are criticizable without being falsifiable.

Recap

The battery B(a)B(a)B(a) assembles everything the system currently demands of an artifact: its own commitments and the criteria of every problem it addresses. Battery-equivalence a′≈Baa' \approx_B aa′≈B​a identifies variants that, from the battery’s perspective, are the same explanation reworded — they agree on every verdict in B(a)B(a)B(a). Criticizability crit(a)\mathrm{crit}(a)crit(a) asks whether aaa has any attack surface at all (I(a)≠∅I(a) \neq \emptysetI(a)=∅). Modifiability mod(a)\mathrm{mod}(a)mod(a) asks whether the variation kernel can produce a genuinely distinguishable variant (Pr⁡a′∼μ(⋅∣a)[a′̸≈Ba]>0\Pr_{a' \sim \mu(\cdot \mid a)}[a' \not\approx_B a] > 0Pra′∼μ(⋅∣a)​[a′≈B​a]>0). An artifact is epistemically active when both hold — it can be attacked and it can be meaningfully varied.

This pair replaces Popperian falsifiability: empirical testing is the special case where some commitment is observation-valued, so mathematics, philosophy, and the arts enter the system on the same footing as empirical conjectures. The two definitions — crit\mathrm{crit}crit and mod\mathrm{mod}mod — and the equivalence relation ≈B\approx_B≈B​ are the foundation that the next module builds the hard-to-vary measure on.
