# Series-Canon — <Series Title>

The **series** consistency contract. Written once by the **Essentials** course at its curriculum
stage (02), **binding** on every sibling mini-course in the series. Where a course's `canon.md` stops
its parallel-authored modules from drifting apart, this file stops the parallel-authored *mini-courses*
from drifting apart: one vocabulary, one set of metaphors, one voice across the whole series. Be
decisive — one call per item, no "TBD"s.

> **Relationship to each member's `canon.md`.** This file governs only the **shared** surface across
> members. Each mini-course still writes its own `runs/<member-id>/02_curriculum/output/canon.md` for
> its internal module consistency. When the two could conflict on a shared concept, the series-canon
> wins; a member's canon may add course-local terms/metaphors freely.
